들어가기

어느날 문득 서비스를 만들어 보고 싶다는 생각이 듬. 대충 아래와 같은 느낌?
  1. 어떤 event가 발생 하면 sensor에서 값을 읽어온다.
  2. 이전 측정 값이랑 비교해서 변동이 있는 경우 server로 올린다.
  3. 서버에 새 update가 있으면 등록 된 사용자에게 push를 사용 하여 알려준다.
  4. 사용자는 앱을 통해 서버에서 현재 값을 읽어 올 수 있다.
그런데 이게 해야 할게 한두개가 아니네? Embedded code도 만들어야 되고, Server도 만들어서 데이터 저장에 계정 관리에 Rest API도 뚫고, App도 만들어서 Server랑 연동도 해야겠고... 이걸 언제 다하지?!

일단 Server를 살 수도 없고, User 관리, DB 관리, 연동 설정 할 자신도 없고... 해서 BaaS를 쓰기로 했다. (BaaS란? [1], [2]) 내가 선택 한 BaaS는 Parse ! Facebook에서 인수 한 바가 있는 BaaS 제공 업체이다.

할일 정리

그럼 Parse를 사용해서 내가 해봐야 하는게 뭔지 생각해보니 다음과 같더라.
  1. Parse에 service를 위한 cloud 공간 생성(App이라고 부름)
  2. User 생성 (User data와 공용 data를 다로 관리 할 수 있는지?)
  3. Embedded에서 Server로 값 update(Raspberry Pi를 사용 할 예정)
  4. Phone으로 Push notification을 보내기 위한 설정(Android device 사용 예정)
  5. Phone에서 서버로 부터 현재 값 확인
제대로 살펴보지도 않고 해보는건데 잘 될런지는 모르겠다.
